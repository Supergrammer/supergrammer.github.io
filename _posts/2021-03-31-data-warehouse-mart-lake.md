# 데이터 웨어하우스(Data Warehouse, DW)

 **데이터 웨어하우스(Data Warehouse, DW)** 는 정보(Data)와 창고(Warehouse)의 의미가 합성되어 만든 단어로, 조직 내에서 분산되어 운영되는 각각의 데이터베이스 관리 시스템들을 효율적으로 통합해 조정, 관리하는 역할을 한다. 데이터 웨어하우스의 구성은 관리 하드웨어, 소프트웨어, 추출, 변환, 정렬 도구, 데이터베이스 마케팅 시스템, 메타 데이터(meta data), 사용자 접근 및 활용 도구 등으로 구성된다.

 데이터 웨어하우스는 의사 결정을 지원하는 데이터의 집합체로 네 가지의 특성을 가지며, 이는 다음과 같다.

- **주제 지향성(Subject-orientation)** : 데이터를 주제별로 구성함으로써 최종 사용자와 전산 처리가 어려운 사람이라도 이해하기 쉬운 형태가 될 수 있다.
  - 지속적인 운영에 포커스를 두는 것이 아닌, 주제에 따른 데이터를 제공하는 것에 의의가 있다.
  - 의사결정을 위한 데이터 모델링과 분석에 중점을 둔다.
  - 의사결정 프로세스에 도움이 되지 않는 항목들을 과감히 제거해 특정 주제에 대한 간단하고 간결한 보기를 제공할 수 있다.

- **통합성(Integration)** : 데이터가 데이터 웨어하우스에 들어갈 때에는 일관적인 형태로 변환된다.
  - 여러 형태의 데이터베이스에서 오는 데이터들을 분석하고 일관적인 공통의 형태를 가지는 데이터로 변환한다.
  - 일관적인 형태로 변환해야 하는 항목에는 네이밍 컨벤션, 형식, 코딩 등이 있다.
  - 이 특성을 통해 효과적인 데이터 분석에 도움이 되고 일관성을 보장할 수 있다.

- **시계열성(Time-variancy)** : 데이터 웨어하우스의 데이터는 일정 기간 동안 정확성을 보장한다.
  - OS 에 비해 시간 범위가 매우 광범위하다는 특징이 있다.
  - 수집된 데이터가 특정 유효 기간으로 인식되어, 일정한 시점의 정보를 제공한다.
  - 명시적, 암시적으로 시간적인 요소를 포함하며, 이런 이유로 한 번 삽입된 데이터는 업데이트되거나 변경할 수 없다.

- **비휘발성(Nonvolatilization)** : 데이터 웨어하우스에 데이터가 적재되면 일괄 처리(batch) 작업에 의한 갱신 외에는 변경이 수행되지 않는 특성을 가진다.
  - 모든 데이터는 읽기 전용이고, 주기적으로 새로고침된다는 특성이 있다.
  - 데이터의 변동은 한번에 이루어지므로 트랜잭션이나 리커버리, 동시성 제어 매커니즘이 필요하지 않다.
  - DW 에서는 데이터의 삭제, 업데이트, 삽입이 무시되며 데이터 액세스와 로딩만 허용된다.

<br>

# 데이터 마트(Data Mart, DM)

 **데이터 마트(Data Mart, DM)** 는 데이터 웨어하우스 환경에서 전사적으로 생성, 관리되는 데이터에 대해 목적에 따라 사용자에게 제공하는 역할을 한다. 데이터 웨어하우스의 범위가 전사적 범위였다면, 데이터 마트는 특정 조직, 팀에서 사용하는 것을 목적으로 한다.

<br>

# 데이터 레이크 (Data Lake, DL)

 이 밖에도 데이터 웨어하우스의 상위 개념인 **데이터 레이크(Data Lake, DL)** 라는 개념이 있는데, 이는 일관적인 형태로 변환하기 이전의 데이터 소스를 그대로 가져와 저장해 다양성을 보존하는 것을 목적으로 한다. 데이터 웨어하우스는 모든 분석 요구에 대해 하나의 스키마를 사용하는데, 이는 대규모 조직에서는 데이터를 분석하기에 적합한 형태가 아니다. 기업은 데이터 레이크를 통해 비용 절감, 데이터 관리 간소화, AI, ML 통합 대비, 분석 속도, 보안 향상 등의 이점을 얻을 수 있다.
